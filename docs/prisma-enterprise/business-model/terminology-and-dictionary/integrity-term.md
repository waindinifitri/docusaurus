# Integrity Term

| **Term**                                 | **Description**                                                                                                                                                                                               |
| ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Aranea**                               | ARANEA is used by analysts to conduct court hearings, covering civil litigation, criminal, bankruptcy, and intellectual property cases across the country.                                                    |
| **Atomic Check**                         | A single, specific piece of information within a type of check, focusing on one particular detail, such as employment at a specific company.                                                                  |
| **Atomic Check to Dispatch**             | A list of atomic checks that needs to be assigned to the screener.                                                                                                                                            |
| **Atomic Check under QC**                | A list of submitted atomic checks that need to be checked by Quality Control (QC).                                                                                                                            |
| **Atomic Check on-the-go**               | A list of atomic checks that have been submitted by the screener with 1, 2, or 3 days remaining and need to be checked by QC as soon as possible.                                                             |
| **Atomic Check Rejected**                | A list of submitted atomic checks that are rejected by QC and need to be revised and resubmitted by the screener.                                                                                             |
| **Queueing Atomic Check**                | A list of atomic checks assigned to the screener but not yet submitted to QC.                                                                                                                                 |
| **Education Verification**               | Solution that defends user institutions' reputation by authenticating certificates and detecting forged documents.                                                                                            |
| **Employment Background Screening**      | A screening process conducted by a company to track employee candidates' backgrounds.                                                                                                                         |
| **Key Account Executive (KAE)**          | A high-level executive responsible for managing relationships with important clients.                                                                                                                         |
| **Know Your Vendor (KYV)**               | Portal application by Integrity Asia for clients to order and request Due Diligence investigation services.                                                                                                   |
| **Knowme**                               | A marketplace where recruiters and candidates can seek jobs and purchase certifications.                                                                                                                      |
| **KnowYourEmployee (KYE)**               | Solution that assists in verifying the professional experience of employees.                                                                                                                                  |
| **KnowYourVendor**                       | Solution that helps clients mitigate supply chain risk by providing a tool for due diligence on third parties.                                                                                                |
| **Log Management**                       | Systematic collection, storage, and analysis of activity logs from various screening processes and systems to ensure transparency and compliance.                                                             |
| **Minerva**                              | Centralizes employee professional experience verification requests, ensuring data privacy and security, covering Education, Employment, and Medical Record Verification.                                      |
| **Monitoring Module**                    | Allows monitoring of system processes, meeting client needs for re-screening checks on candidates scheduled monthly, yearly, or customized.                                                                   |
| **Multilingual Module**                  | Enables users to select their preferred language in the Prisma Application, enhancing accessibility and user experience.                                                                                      |
| **Multiple Deadlines**                   | Feature in Prisma Application allowing distinct deadlines for various checks, managed by the Sales team during client contract creation.                                                                      |
| **Netforce Global**                      | Company that provides a third-party application for Integrity.                                                                                                                                                |
| **Notification Module**                  | Manages and delivers notifications to users about significant actions or updates within the application.                                                                                                      |
| **Prevention, Detection, Investigation** | Prevention involves stopping issues before they occur, detection identifies issues as they happen, and investigation addresses issues after they've occurred.                                                 |
| **Prisma**                               | Personnel Research Collecting Information Securely by using Multiple and Accurate sources, an Integrity Platform that utilizes the user behavior needed based on the implemented value.                       |
| **QC Module Assessment**                 | Quality Control process designed to provide a scoring assessment by Quality for any checks submitted by the Screener.                                                                                         |
| **Qualification Place**                  | Place or location where an individual has worked or obtained their qualifications, including institution names.                                                                                               |
| **Qualifying Process**                   | Process of ensuring prospects are a good fit for the solution, conducted by the KAE team using a Qualifier Account.                                                                                           |
| **Retention Module**                     | Manages and controls the retention of collected data in Prisma, specifying how long different types of data will be retained before deletion.                                                                 |
| **Sales Module**                         | Streamlines client onboarding, initiated by emailing the sales team at screening-asia.com to determine account type (Limited Basic or Full Access).                                                           |
| **Screening Package**                    | Process of evaluating individuals or information to determine suitability, eligibility, or relevance.                                                                                                         |
| **OCR**                                  | System integration in Prisma that automatically generates candidate forms from scanned CVs/documents, validated by KAE before screening by operations.                                                        |
| **Screening Request Form**               | Form used to request a screening.                                                                                                                                                                             |
| **Telephonic Module**                    | Module related to or conveyed by telephone.                                                                                                                                                                   |
| **Tenant**                               | A person or user with credentials for projects, with the definition depending on context.                                                                                                                     |
| **Tolerance Matrix**                     | A structured framework used by Integrity Asia to categorize the results of various checks conducted during a background check process. It is used for determining the status of a verification/search result. |
| **Type Of Check**                        | Specific category of information verified during the background check process, such as employment check, education check, ID check, etc.                                                                      |
| **Type Of Check Category**               | Classification system for organizing types of checks based on the method of data collection, such as "Search", "Verification," and "Hybrid".                                                                  |
| **Screening Report**                     | Document compiling the results of the background check, including discrepancies and other relevant data.                                                                                                      |
| **Interim Report**                       | A screening report published before all checks are completed.                                                                                                                                                 |
| **Final Report**                         | A screening report published after all checks are completed.                                                                                                                                                  |
| **Notification**                         | Communication alert informing users about updates, actions required, or results.                                                                                                                              |
| **Contract**                             | Legally binding agreement outlining terms, conditions, scope of services, pricing, and responsibilities of both parties.                                                                                      |
| **Turnaround Time (TAT)**                | Total time taken from initiating a background check to delivering the results to the client.                                                                                                                  |
| **Methodology**                          | Specific procedures, processes, and requirements for conducting background checks as requested by the client.                                                                                                 |
| **Invoice**                              | Document detailing services rendered and amount due, including itemized charges and payment terms.                                                                                                            |
| **Client Company**                       | Company belonging to the client with a contract and registered on the platform.                                                                                                                               |
| **Client Account**                       | Account registered as a client on the platform.                                                                                                                                                               |
| **Client User**                          | Represents the client organization or employer utilizing the background screening services.                                                                                                                   |
| **Client Admin**                         | Oversees client relationships, requirements, and service delivery on behalf of the client's organization.                                                                                                     |
| **Client Accountant**                    | Responsible for financial aspects, such as billing inquiries and payment.                                                                                                                                     |
| **Client Auditor**                       | Conducts internal audits or reviews related to screening activities.                                                                                                                                          |
| **Vendor**                               | Partner organization providing background screening services in a specific country.                                                                                                                           |
| **Service Provider**                     | Partner organization providing specific services or resources supporting the background check process, integrated with the platform using API.                                                                |
| **Freelancer**                           | Individual providing background screening services on a freelance or contract basis.                                                                                                                          |
| **Field Agent**                          | Responsible for conducting verifications on background checks from the field.                                                                                                                                 |
| **Investigator**                         | Conducts in-depth investigations or research on individuals or entities from the field.                                                                                                                       |
| **Surveillance**                         | Monitors and observes individuals or locations for specific purposes, such as risk assessment or due diligence.                                                                                               |
| **Due Diligence Operator**               | Responsible for processing due diligence requests from clients.                                                                                                                                               |
| **Applicant**                            | Individual subject to a background screening process initiated by a client organization/employer.                                                                                                             |
| **Guest**                                | User with limited access to the platform, such as potential clients.                                                                                                                                          |
| **Business Administrator**               | Manages overall business operations and strategies related to the screening industry.                                                                                                                         |
| **Production Administrator**             | Oversees day-to-day operations within the platform, such as monitoring screening and screeners.                                                                                                               |
| **Candidate**                            | Individual subject to a background screening process initiated by a client organization or employer.                                                                                                          |
| **Screener**                             | Responsible for conducting verifications on background checks, including VSP Operators and Field Agents.                                                                                                      |
| **Screening**                            | The process of conducting various checks and verifications on an individual's background.                                                                                                                     |
| **Screening to Identify**                | A list of screenings to be identified by KAE to start the background check process.                                                                                                                           |
| **Screening Ongoing**                    | A list of screenings still in progress.                                                                                                                                                                       |
| **Screening to Publish**                 | A list of screenings that need to be published to the client as a report.                                                                                                                                     |
| **Screening ID**                         | A unique number assigned by the system to identify each screening.                                                                                                                                            |
| **Screening on-the-go**                  | Feature for tracking screening progress in real-time.                                                                                                                                                         |
| **Type of Request**                      | Method by which a background check is initiated, such as "Invitation Request" or "Direct Request."                                                                                                            |
| **Invitation Request**                   | Request type where the client invites the candidate to submit data and fill out the application form via the platform.                                                                                        |
| **Direct Request**                       | Request type where the client submits the candidate's documents for the background check process.                                                                                                             |
